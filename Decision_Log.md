# Decision Log

## Architecture & Tech Stack

**Language:** Python
- **Reasoning:** Industry standard for data analysis, BI tools, and LLM integrations. Extensive libraries for interacting with APIs, file parsing (pandas), and building AI agents.

**UI Framework:** Streamlit
- **Reasoning:** Extremely fast to prototype and host interactive data apps. Native support for conversational interfaces (`st.chat_message`), easy to display tables, metrics, and markdown for tool-call traces. It requires zero setup by the evaluator once hosted on Streamlit Community Cloud.

**LLM & Agent Framework:** LangChain / OpenAI (GPT-4o)
- **Reasoning:** LangChain provides robust tool-calling (function calling) capabilities necessary for agentic workflows. OpenAI models are state-of-the-art for reasoning, generating code/queries, and responding in the strict format requested by the assignment (Insight Summary, Key Metrics, etc.). 

**Data Processing:** pandas
- **Reasoning:** Unmatched for tabular data manipulation. Perfect for normalizing messy data fields, handling missing values natively (`.fillna()`, `.dropna()`), and joining datasets (matching Deals to Work Orders) to answer cross-board queries.

**Monday.com Integration:** Custom Python API Wrapper (GraphQL via Requests)
- **Reasoning:** The assignment enforces live API data fetching without preloading or caching. A direct GraphQL API wrapper using `requests` ensures we grab the exact state of the boards during the query. It allows us to implement pagination efficiently.

## Handling Data Resilience & Inconsistencies
1. **Normalization:** Convert all currency to standard numeric format, standardize sector names (e.g., lowercasing, stripping whitespace), and map stages to a unified enum.
2. **Missing Values:** Fill missing critical numeric fields with 0 or exclude them from averages. Missing categorical fields will be labeled as "Unknown".
3. **Data Quality Caveats:** We will explicitly instruct the LLM to read the data cleaning logs generated by the pandas module and append a "Data Quality Notes" section to its response, alerting the founder if major data was missing from the analysis.

## Agent Visibility
Streamlit's `st.status()` or custom expanders will be used to log and display the agent's thought process and API calls in real-time, fulfilling the requirement for a "visible action/tool-call trace."
